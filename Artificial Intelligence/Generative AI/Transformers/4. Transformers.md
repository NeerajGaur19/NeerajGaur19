Transformers are deep learning models introduced in the paper "Attention Is All You Need" (2017). 
They rely entirely on the Attention Mechanism—no RNNs or LSTMs—which allows massive parallelization and better long-range dependency learning.

## Key Features

* No recurrence or convolution
* Entirely based on Self-Attention
* Captures long-range dependencies efficiently
* Highly parallelizable (faster training)
* Scales well with large datasets and compute
 
## Core Idea

Each word in a sequence looks at all other words at once to decide how much attention to pay to them, and builds a rich representation.

## Building Blocks

### 1. Input Embedding

Convert tokens to vectors.

### 2. Positional Encoding

Add position information (since there is no recurrence).

### 3. Multi-Head Self-Attention

Compute attention in parallel using multiple attention heads.

### 4. Feed Forward Network (FFN)

Two linear layers with a non-linearity.

### 5. Add & Norm

Residual connection + Layer Normalization.

### 6. Masked Multi-Head Attention (in Decoder)

Prevents looking at future tokens.

# Trasformers Encoder

Suppose the input sentence is:

    I love AI

The encoder's job is: Convert this sentence into context-aware vectors that capture the meaning of every word. These vectors are then sent to the decoder (for translation) or to a classifier (for BERT).

## Step 1: Tokenization

Sentence

    I love AI

Tokenizer converts words into IDs.

    I      → 25
    
    love   → 98
    
    AI     →150

Result

    [25,98,150]

Computers cannot understand words.

They understand numbers.

## Step 2: Input Embedding

Each token ID becomes a dense vector.

Suppose embedding dimension = 4.
    
    Word	      Embedding
    I	        [0.2,0.8,0.4,0.7]
    love	    [0.6,0.1,0.5,0.3]
    AI	      [0.9,0.7,0.2,0.8]

These vectors contain semantic meaning.

Example

    King and Queen
    ↓
    Similar vectors

## Step 3: Positional Encoding

Attention alone doesn't know word order.

These two sentences

    Dog bites man
    
    Man bites dog

contain the same words.

Without position information, the model thinks they are almost identical. So we add positional vectors.

Example

Position	          Vector
  0	            [0.1,0.2,0.3,0.4]
  1	            [0.5,0.6,0.7,0.8]
  2	            [0.9,1.0,1.1,1.2]

Now

    Embedding
    +
    Position
    =
    New Input

Example

    Word
    
    [0.2 0.8 0.4 0.7]
    +
    Position
    [0.1 0.2 0.3 0.4]
    =
    [0.3 1.0 0.7 1.1]

Now the model knows

* what the word is
* where it occurs

## Step 4: Multi-Head Self-Attention

This is the heart of Transformers.

Each word asks

    Which other words are important for me?

For our sentence

    I love AI

Attention might learn

    I
    ↓
    love
    
    AI

Meaning

"I" attends strongly to "love".

Love attends strongly to AI.

AI attends to love.

Each word collects useful information from every other word.

Output

    Old embedding
    ↓
    New contextual embedding

Now "AI" knows it is the object of "love."

## Step 5: Residual Connection (Add)

Instead of replacing the original embedding,

we add it.

    Original Input
    
    +
    
    Attention Output
    
    =
    
    Combined Output

Example

Original

    [3,2]

Attention

    [1,1]

Output

    [4,3]

Why?


Because the original embedding already contains useful information. Attention only adds extra context.


## Step 6: Layer Normalization

Now normalize the values.

Suppose

    [100
    120
    90
    80]

becomes approximately

    [0.5
    1.1
    -0.3
    -1.3]

Benefits

* stable training
* faster convergence
* prevents exploding activations


## Step 7: Feed Forward Network (FFN)

Every word now goes independently through a small neural network.

Structure

    Linear
    
    ↓
    
    ReLU (or GELU)
    
    ↓
    
    Linear

Example

Input

    [4.2,2.8]

↓

Linear

↓

ReLU

↓

Linear

↓

Output

    [5.3,3.1]

Unlike attention, FFN does not mix information between words. It refines each word's representation independently.


## Step 8: Residual Connection Again

Again,

    FFN Output
    +
    Previous Input

This helps

* preserve information
* improve gradient flow
* train deep networks


## Step 9: Layer Normalization Again

Normalize once more.

Output becomes the encoder output.

# Transformer Decoder

The decoder's job is to generate the next word (or token) one token at a time.

Unlike the encoder, which can see the entire input sentence, the decoder cannot see future words. It only sees:

1. The words generated so far.
2. The encoder's output (if using an encoder-decoder model like the original Transformer, T5, or BART)

Each decoder layer contains:

1. Masked Multi-Head Self Attention
2. Cross Attention (Encoder-Decoder Attention)
3. Feed Forward Network (FFN)

Each sub-layer is followed by:

* Residual Connection
* Layer Normalization

## Step 1 — Input Embedding

Suppose vocabulary IDs are

    <START> = 1
    J'aime = 15
    l'Inde = 29

Embedding dimension = 512

Embedding table

    1  → [0.4,0.2,...]
    15 → [0.8,0.1,...]
    29 → [0.3,0.5,...]

The decoder converts IDs into vectors.

## Step 2 — Positional Encoding

Word order matters.

The decoder adds positional vectors.

    Embedding
    +
    Position Encoding
    =
    Final Input

Exactly like the encoder.


## Step 3 — Masked Self Attention

This is the biggest difference from the encoder.

Suppose during training we have

    <START>
    J'aime
    l'Inde

The decoder must not cheat by looking ahead.

Instead it uses a mask.

Normal attention matrix

              START  J'aime  l'Inde
    
    START       ✓      ✓        ✓
    
    J'aime      ✓      ✓        ✓
    
    l'Inde      ✓      ✓        ✓

Masked attention

              START  J'aime  l'Inde
    
    START       ✓      X        X
    
    J'aime      ✓      ✓        X
    
    l'Inde      ✓      ✓        ✓

So:

* START sees only START.
* J'aime sees START and J'aime.
* l'Inde sees everything before it.

Future tokens are hidden.

This is called causal masking or look-ahead masking.

# Why masking?

Imagine predicting

    I love India

If the decoder could already see

    India

while predicting

    love

training would become trivial and the model would not learn to predict the next token.

The mask prevents that.

## Step 4 — Compute Q, K, V

Exactly like encoder.

Each token vector becomes

    Q = XWQ
    
    K = XWK
    
    V = XWV

Suppose

    J'aime
    
    ↓
    
    Embedding
    
    ↓
    
    Q
    K
    V

Every decoder token gets Q, K and V.

## Step 6 — Multi-Head Attention

Instead of one attention,

the decoder computes several in parallel.

Example

    Head 1
    Focuses on grammar
    
    Head 2
    Focuses on subject
    
    Head 3
    Focuses on tense
    
    Head 4
    Focuses on long-distance dependency

Outputs are concatenated.

## Step 7 — Residual + LayerNorm

Same as encoder.

    Output
    
    +
    
    Original Input
    
    ↓
    
    LayerNorm

This stabilizes training.

## Step 8 — Cross Attention

Now comes the second attention block.

Instead of attending to decoder words,

the decoder attends to the encoder output.

Suppose

Encoder input

    I love India

Encoder output

    H1
    H2
    H3

Decoder currently has

    J'aime

The decoder creates:

* Query (Q) from the decoder state.
* Keys (K) from the encoder output.
* Values (V) from the encoder output.

        Decoder State
        
        ↓
        
        Query
        
        Encoder Output
        
        ↓
        
        Keys
        
        ↓
        
        Values

Then

    Attention(Q,K,V)

This lets the decoder decide which source words are most relevant while generating the next token.

For example, when generating "Inde", the decoder may attend strongly to the encoder representation of "India".

## Step 10 — Repeat Decoder Layers

One decoder layer finishes.

The output becomes input to the next decoder layer.

Example

    Decoder Layer 1
    
    ↓
    
    Decoder Layer 2
    
    ↓
    
    Decoder Layer 3
    
    ↓
    
    ...
    
    ↓
    
    Decoder Layer 6

(The original Transformer used 6 decoder layers. Modern models often use many more.)

## Step 11 — Linear Layer + Softmax

After the final decoder layer, each token representation is projected to the vocabulary size.

For a vocabulary of 50,000 words:

    512
    
    ↓
    
    Linear
    
    ↓
    
    50000 logits
    
    ↓
    
    Softmax
    
    ↓
    
    Probability distribution

Example:

    J'aime   0.01
    
    le       0.03
    
    l'Inde   0.82
    
    Paris    0.001

The highest-probability token is selected (or sampled, depending on the decoding strategy).

## Step 12 — Autoregressive Generation

At inference, the decoder repeats this process:

    Input: <START>
    Predict J'aime
    Input: <START> J'aime
    Predict l'
    Input: <START> J'aime l'
    Predict Inde
    Continue until an <END> token is produced.

This left-to-right generation is called autoregressive decoding.
