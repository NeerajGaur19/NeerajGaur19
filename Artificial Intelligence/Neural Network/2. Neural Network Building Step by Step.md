# 2. Hidden Layer

Let's make Hidden layer next.

We can take as many as nodes of our choice in a layer. There is no restriction on nodes. But the optimized number of nodes depend on the complexity of data. 
More the number of nodes, more will be processing time.

Also, we can take as many number of hidden layers in neural network. The more hidden layers will take the more deeper neural network becomes (Deep learning).
More the number of layers, more will be processing time and high chances of overfitting (learn the data instead of pattern).

#### Now after weights and bias is calculated for all the rows of data then we take average gradient of all graident values.

For example if we have batch_size of 10 for 300 rows of data then each of 30 batch will have 10 rows of data.
* We calculate gradient for each row of data in batch 1.
* We take average of graident by dividing by 10. 
* Now the updated weights and bias values using average graident, will be used by batch2 for training during forward propagation.
* Same process repeats till batch 30.

If Epoch is also used lets say epoch = 50, then all 300 rows will be given to model 50 times. 
<br>This will help weights and bias more closer and error will decrease.
