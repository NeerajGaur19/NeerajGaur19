# Principal Component Analysis (PCA) – Detailed Explanation

PCA is one of the most important dimensionality reduction techniques in Machine Learning.

### Its goal is to: Reduce the number of features while preserving as much information (variance) as possible.

## The Problem PCA Solves  

Imagine a dataset with:

<img width="622" height="268" alt="image" src="https://github.com/user-attachments/assets/4bbb928f-5d8b-4d2d-9d1a-c4b0d9bcf934" />

Notice that:

* Taller people tend to weigh more.
* Height and Weight are highly correlated.
* Both features contain similar information.

Instead of keeping both features, PCA tries to combine them into fewer variables called Principal Components.

## What is a Principal Component?

A Principal Component is a new feature created from the original features.

These new features:

* Are combinations of original features
* Capture maximum variance
* Are uncorrelated with each other

For example:

* PC1=0.7×Height+0.7×Weight
* PC2=0.7×Height−0.7×Weight

PC1 captures most of the information. <br>
PC2 captures the remaining information.

Often, PC1 alone may explain 90% + of the variance.

# PCA Mathematics

The principal components come from the eigenvectors of the covariance matrix.

The transformation can be represented as:

Z=XW

Where:

* X = standardized data
* W = matrix of selected eigenvectors
* Z = transformed data (principal components)
<br> You don't need to compute this manually in practice.


## Explained Variance Ratio

One of the most important outputs.

### Example:

      from sklearn.decomposition import PCA
      
      pca = PCA()
      pca.fit(X_scaled)

      print(pca.explained_variance_ratio_)

### Output:

[0.75, 0.15, 0.07, 0.03]

## Choosing Number of Components

### Method 1: Explained Variance

Keep enough components to explain:

* 90%
* 95%
* 99%

of the variance.

      pca = PCA(n_components=0.95)

This automatically keeps enough components to retain 95% variance.

### Method 2: Scree Plot

      import matplotlib.pyplot as plt
      
      plt.plot(range(1, len(pca.explained_variance_ratio_)+1),
               pca.explained_variance_ratio_.cumsum())

Choose the "elbow point" where additional components add little information.

## PCA in Scikit-Learn

      from sklearn.preprocessing import StandardScaler
      
      from sklearn.decomposition import PCA
      
      # Scale
      
      scaler = StandardScaler()
      
      X_scaled = scaler.fit_transform(X)
      
      # PCA
      
      pca = PCA(n_components=2)
      
      X_pca = pca.fit_transform(X_scaled)

### Now:

      print(X.shape)

### Output:

      (1000, 20)

### After PCA:

      print(X_pca.shape)

### Output:

      (1000, 2)

20 features become 2.

## Advantages of PCA

✅ Reduces dimensionality

✅ Faster model training

✅ Removes multicollinearity

✅ Helps visualization

✅ Reduces storage requirements

✅ Removes some noise


## Disadvantages of PCA

❌ Components are hard to interpret

Instead of:

* Age
* Salary

You get:

* PC1
* PC2

which have no direct business meaning.

❌ Information loss may occur.

❌ Assumes linear relationships.

❌ Sensitive to feature scaling.

PCA can be used across paradigms, but its role changes depending on whether you are working with supervised or unsupervised data.
Because PCA itself doesn't look at target labels ($y$), it is always an unsupervised step, but its outputs are incredibly useful for boosting the performance of supervised models.

Here is how PCA is applied in each scenario:

## 1. PCA in Unsupervised Learning

This is PCA’s natural habitat. Since there are no labels, PCA is used purely to understand the structure of the data.

### Dimensionality Reduction: 

If you have 100 features, PCA compresses them into a few principal components while keeping most of the information (variance).

### Data Visualization: 

Humans can't visualize 10D or 100D space. PCA reduces the data to 2D or 3D so you can plot it and look for natural clusters, patterns, or extreme outliers.

### Pre-processing for Clustering: 

Algorithms like K-Means struggle in high dimensions due to the "curse of dimensionality" (distances between points start to look the same). Running PCA before K-Means helps the clustering algorithm perform much better and run faster.

## 2. PCA in Supervised Classification

In classification, you are predicting a categorical label (e.g., Spam vs. Not Spam, Tumor vs. Benign).

### How it's applied: 
You run PCA on your feature matrix ($X$) to create a smaller set of principal components. You then feed these components into a classifier like Logistic Regression, Support Vector Machines (SVM), or Random Forests.
      
### Why do it? * Prevents Overfitting: 
If you have more features than data points ($n \ll p$), models overfit easily. PCA reduces the feature count to prevent this.

      Fixes Multicollinearity: Classifiers can get confused when features are highly correlated. Because principal components are perfectly perpendicular
      (orthogonal) to each other, multicollinearity is completely eliminated.

## 3. PCA in Supervised Regression

In regression, you are predicting a continuous numerical value (e.g., House Prices or Stock Value).

## Principal Component Regression (PCR): 
This is a formal technique where you apply PCA to your features and then run a Linear Regression on the resulting components.

## Why do it? 
Much like in classification, if your independent variables are highly correlated (e.g., a house's "square footage" and "number of rooms"), standard Linear Regression estimates become highly unstable. PCR solves this by making the predictors completely independent of one another.

## Note: The Golden Rule: Don't Leak Data!

When using PCA for supervised learning (Classification or Regression), always fit the PCA transformer on the training set only, and then transform both the training and testing sets. 
If you run PCA on the entire dataset before splitting, information from the test set will "leak" into your training data, giving you overly optimistic (and fake) performance metrics.
