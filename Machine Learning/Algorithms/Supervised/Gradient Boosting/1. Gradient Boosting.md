Gradient Boosting is a supervised ensemble learning algorithm that combines many weak learners (usually small decision trees) sequentially. Each new tree is trained to correct the mistakes (errors or residuals) made by the previous trees.

Think of it like a student solving a math problem.

    First attempt → several mistakes.
    Teacher points out the mistakes.
    Student corrects those mistakes.
    Teacher points out the remaining mistakes.
    Student corrects them again.

After several corrections, the solution becomes very accurate.

Gradient Boosting works exactly like this.


## Step 1: Training Data

Suppose we want to predict house prices.

    House	Area (sq ft)	Actual Price (₹ Lakhs)
    A	        1000	            50
    B	        1200	            60
    C	        1500	            80
    D	        1800	            95

Our target is:

    y = House Price


## Step 2: Build the First Decision Tree

Instead of building a huge tree, Gradient Boosting starts with a small decision tree.

Suppose it predicts:
    
    House	Actual	Predicted
    A	      50	    48
    B	      60	    58
    C	      80	    75
    D	      95	    90

Notice it isn't perfect.


## Step 3: Calculate Errors (Residuals)

Now calculate:

Residual = Actual − Predicted
    
    House	    Actual	Predicted	Residual
    A	            50	    48	        +2
    B	            60	    58	        +2
    C	            80	    75	        +5
    D	            95	    90	        +5

These residuals represent what the first model still needs to learn.


## Step 4: Train the Second Tree

The second tree is not trained on the original house prices.

Instead, it learns the residuals:
    
    House	Residual
    A	        +2
    B	        +2
    C	        +5
    D	        +5

Suppose it predicts:
    
    House	Residual	Predicted Residual
    A	        +2	        +1.8
    B	        +2	        +2.0
    C	        +5	        +4.7
    D	        +5	        +5.0


## Step 5: Update the Predictions

Instead of replacing the old prediction, Gradient Boosting adds a correction.

Formula:

    New Prediction
    =
    Old Prediction
    +
    Learning Rate × Residual Prediction

Suppose the learning rate is 0.1.

For House A:

    48 + 0.1 × 1.8
    
    = 48.18

For House C:

    75 + 0.1 × 4.7
    
    = 75.47

The predictions move closer to the true values.

## Step 6: Calculate New Residuals

Now compute the remaining errors.
    
    House	    Actual	        New Prediction	    New Residual
    A	        50	                48.18	            1.82
    B		    60		            58.20		        1.80
    C		    80		            75.47		        4.53
    D		    95		            90.50		        4.50

The errors are now smaller.


## Step 7: Train Another Tree

Tree 3 learns these new residuals.

Again,

    Prediction
    
    ↓
    
    Residual
    
    ↓
    
    New Tree
    
    ↓
    
    Correction
    
    ↓
    
    Better Prediction
    
Repeat this process many times.


## Final Model

Suppose we train five trees.

    Tree 1
         ↓
    Tree 2
         ↓
    Tree 3
         ↓
    Tree 4
         ↓
    Tree 5

Final prediction:

    Prediction
    
    =
    
    Tree1
    
    +
    
    0.1(Tree2)
    
    +
    
    0.1(Tree3)
    
    +
    
    0.1(Tree4)
    
    +
    
    0.1(Tree5)

Every tree contributes a small correction.

## Why is it Called "Gradient" Boosting?

The algorithm tries to minimize a loss function.

For regression, a common loss is Mean Squared Error (MSE):

<img width="200" height="85" alt="image" src="https://github.com/user-attachments/assets/bd805766-b980-4930-8f6c-4ad7efde69dc" />

Gradient Boosting improves predictions by moving in the direction that reduces this loss most quickly, using the negative gradient of the loss function.

This is where the word Gradient comes from.

## Why "Boosting"?

Because it boosts many weak learners into one strong learner.

Example:

    Weak Tree 1
    
    +
    
    Weak Tree 2
    
    +
    
    Weak Tree 3
    
    +
    
    Weak Tree 4
    
    ↓
    
    Strong Model

## Learning Rate

Very important interview topic.

    New Prediction
    
    =
    
    Old Prediction
    
    +
    
    Learning Rate × Tree Output

If:

    Learning Rate = 1

Large corrections are made.

If:

    Learning Rate = 0.1

## Small corrections are made.

Small Learning Rate

    ✔ Better generalization
    
    ✔ Lower chance of overfitting
    
    ❌ Needs more trees
    
    Large Learning Rate
    
    ✔ Faster learning
    
    ❌ Higher chance of overfitting



## Important Hyperparameters

### 1. n_estimators

Number of trees.

Example:

    GradientBoostingRegressor(
        n_estimators=100
    )


### 2. learning_rate

How much each tree contributes.

Typical values:

    0.01
    
    0.05
    
    0.1

### 3. max_depth

Depth of each decision tree.

Usually:

    3
    
    4
    
    5

Gradient Boosting generally uses shallow trees.

### 4. subsample

Use only part of the data for each tree.

Helps reduce overfitting.
