# Ensemble Learning Algorithm

An Ensemble Learning Algorithm is a machine learning technique where multiple models are combined to produce a better prediction than a single model.

# Random Forest

Random Forest is an ensemble learning algorithm that combines multiple Decision Trees and makes predictions by majority voting.

# Why was Random Forest created?

A single Decision Tree:

## Advantages

* Easy to understand
* Handles nonlinear relationships
* No scaling required

## Problem

Decision Trees often overfit.

Example:

* Training Accuracy = 100%
* Test Accuracy = 75%

The tree memorizes the training data instead of learning general patterns.

Random Forest solves this by combining many trees.

## Core Idea

Instead of asking one expert (one tree):

    Doctor A → Disease A

Ask 100 experts:

    Doctor A → Disease A
    Doctor B → Disease A
    Doctor C → Disease B
    Doctor D → Disease A
    ...

Final answer:

    Disease A (majority vote)

This reduces variance and improves generalization.

# How Random Forest Works

## Step 1: Bootstrap Sampling

Suppose we have 1000 records.

For Tree 1:

    Randomly select 1000 rows WITH replacement

For Tree 2:

    Randomly select another 1000 rows WITH replacement

And so on.

Because sampling is done with replacement, some rows appear multiple times while some are not selected.

Example:

Original:

    1 2 3 4 5

Bootstrap sample:

    1 2 2 4 5

Record 3 is missing and 2 appears twice.

This technique is called Bagging (Bootstrap Aggregating).

## Step 2: Random Feature Selection

Assume 10 features:

    Age
    Salary
    BMI
    Weight
    Height
    Gender
    Smoking
    City
    Exercise
    BloodPressure

A normal Decision Tree evaluates all 10 features.

Random Forest chooses a random subset.

Example:

Tree 1 sees:

    Age
    BMI
    Smoking

Tree 2 sees:

    Salary
    Weight
    Exercise

Tree 3 sees:

    Height
    Gender
    City

This creates diversity among trees.

## Step 3: Build Multiple Trees

    Tree 1
    Tree 2
    Tree 3
    ...
    Tree 100

Each tree:

* Gets different samples
* Gets different features

Therefore each tree learns slightly different patterns.

## Step 4: Majority Voting

Suppose a new patient arrives.

Predictions:
    
    Tree 1 → Flu
    Tree 2 → Flu
    Tree 3 → Dengue
    Tree 4 → Flu
    Tree 5 → Flu

Vote count:
    
    Flu = 4
    Dengue = 1

Final Prediction:

    Flu

# Why is it called "Random" Forest?

Two sources of randomness:

## 1. Random rows

       Bootstrap Sampling
  
## 2. Random features

       Random Feature Selection

Together they create many different trees.

# Important Hyperparameters

## n_estimators

Number of trees. How many decision trees should Random Forest build?

Syntax    :     RandomForestClassifier(n_estimators=100)

if n_estimators = 100, then 100 different decision trees are built.

More trees generally improve stability but increase training time.

## max_depth

Maximum depth of each tree. How deep each decision tree is allowed to grow. This controls overfitting.

Syntax : RandomForestClassifier(max_depth=5)

Imagine a tree. Without restriction

<img width="252" height="208" alt="image" src="https://github.com/user-attachments/assets/79abb449-a42c-4877-be2b-8de0a8268104" />

The tree keeps growing.

If max_depth = None, then No limit. Tree grows until it cannot split anymore.

If max_depth = 5, then Maximum depth = 5. Tree stops growing after level 5.

    Root
     |
     |---- Age
          |
          |---- Income
                 |
                 |---- Balance
                        |
                        |---- Stop

## min_samples_split

The minimum number of samples that a node must contain before the algorithm is allowed to try splitting it.

Syntax : RandomForestClassifier(min_samples_split=10)

## criterion

This determines how the tree decides the best split at each node.

There are two common choices:

### Gini Impurity

Measures how mixed the classes are.

Example

    100 students
        
    100 Pass
        
    Pure node
        
    Gini = 0

Another node
    
    50 Pass
    50 Fail
    
    Highly mixed
    
    Higher Gini

The algorithm chooses the split that reduces impurity the most.

### Entropy

Based on information theory.

It measures the amount of disorder or uncertainty in the node.

* Lower entropy = purer node.
* Higher entropy = more mixed classes.

The algorithm selects the split that gives the largest reduction in entropy (highest information gain).

Features considered at each split.

    max_features='sqrt'

Common default for classification.

# Feature Importance

Random Forest can tell which features are most useful.

Example:

    model.feature_importances_

Output:

    Age            0.40
    BMI            0.25
    Smoking        0.20
    Exercise       0.10
    Gender         0.05

Interpretation:

    Age is most influential
    Gender is least influential

# Advantages

1. High Accuracy

       Usually better than a single Decision Tree.

2. Less Overfitting

       Many trees reduce variance.

3. Handles Nonlinearity

       Can learn complex patterns.

4. Handles Missing Relationships

       No assumption of linearity.

5. No Feature Scaling Required

        Works directly on raw numerical features.

        Not required:
        StandardScaler
        MinMaxScaler

# Disadvantages

1. Slower than Decision Tree

        Many trees must be built.

2. Less Interpretable

        A single tree can be visualized.

        100 trees cannot easily be explained.

3. Larger Memory Usage

       Stores many trees.

# Time Complexity Intuition 

Decision Tree

    One tree
    Fast

Random Forest

    100 trees
    More computation
    Better accuracy

Trade-off:
    
    Speed ↓
    Accuracy ↑
